# Evolução — Disponibilidade mobile e agenda compacta

## Decisões aprovadas

- O sistema é para locação/aluguel, não venda.
- A barra inferior deve priorizar o uso real no celular.
- Disponibilidade precisa estar na barra inferior.
- Agenda deve mostrar somente o essencial: hora, cliente e item/equipamento.
- Detalhes ficam escondidos; o usuário toca na linha para abrir a reserva.
- Disponibilidade considera a quantidade cadastrada menos os itens já alugados naquela data.
- Ao tocar em um item, o sistema mostra onde ele está naquela data: horário, cliente e bairro.
- Análise automática de rota fica para etapa futura.

## Entregas realizadas

- Nova tela `/painel/disponibilidade`.
- Novo cálculo simples de disponibilidade por data.
- Detalhe do item com locações do dia e bairros.
- Agenda compacta.
- Barra inferior reorganizada:
  - Início
  - Agenda
  - Disponível
  - Cliente
  - Contrato
- Linguagem ajustada para locação/aluguel.
